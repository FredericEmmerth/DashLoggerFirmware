\doxysubsection{Detailed Description}
This Struct defines a Signal. Every Signal should have a friendly Name, to be easily identified by a human. Every Signal has to have a \mbox{\hyperlink{structsignals__signal__struct_ad30a372a63b611762abe1c5e0c5fa6aa}{signals\+\_\+signal\+::type}} and a \mbox{\hyperlink{structsignals__signal__struct_af68804a49ea40ef5000c9f76ab47d2d8}{signals\+\_\+signal\+::data\+\_\+type}}. Depending on the \mbox{\hyperlink{structsignals__signal__struct_ad30a372a63b611762abe1c5e0c5fa6aa}{signals\+\_\+signal\+::type}} and \mbox{\hyperlink{structsignals__signal__struct_af68804a49ea40ef5000c9f76ab47d2d8}{signals\+\_\+signal\+::data\+\_\+type}} other members of the struct have to be defined. If a signal is of type SIGNALS\+\_\+\+CAN\+\_\+\+MESSAGE, the id and the interface\+\_\+number have to be defined. The Signal can be either of data\+\_\+type SIGNALS\+\_\+\+FLOAT\+\_\+\+SIGNAL, SIGNALS\+\_\+\+UINT32\+\_\+\+T\+\_\+\+SIGNAL or SIGNALS\+\_\+\+STRING\+\_\+\+SIGNAL. The corresponding callback function can\+\_\+convert\+\_\+float, can\+\_\+convert\+\_\+uint32\+\_\+t or can\+\_\+convert\+\_\+string has to be defined.

If a signal is of type SIGNALS\+\_\+\+INTERNAL\+\_\+\+SIGNAL, it can be of data\+\_\+type SIGNALS\+\_\+\+FLOAT\+\_\+\+SIGNAL, SIGNALS\+\_\+\+UINT32\+\_\+\+T\+\_\+\+SIGNAL or SIGNALS\+\_\+\+STRING\+\_\+\+SIGNAL. The corresponding callback function internal\+\_\+convert\+\_\+float, internal\+\_\+convert\+\_\+uint32\+\_\+t or internal\+\_\+convert\+\_\+string has to be defined.

If a signal is of type SIGNALS\+\_\+\+DISPLAY\+\_\+\+SIGNAL, it\textquotesingle{}s data\+\_\+type has to be SIGNALS\+\_\+\+STRING\+\_\+\+SIGNAL, and its callback function internal\+\_\+convert\+\_\+string has to be defined in a way to produce correct Commands for the Display to be interpreted. Signals of this Type are read by \mbox{\hyperlink{group___c_o_m_m_a_n_d}{COMMAND}} to be sent of to the Display using the \mbox{\hyperlink{group___s_h_o_r_t_p_r_o_t_o_c_o_l}{SHORTPROTOCOL}}.

A Signal is therefore defined by its type and data\+\_\+type.
