\doxysubsection{Detailed Description}
This Module Handles the Information Interpretation Handling of All Inputs to Outputs. Signals have a Type, wich changes how it is interpreted and used. For more on this, refer to \mbox{\hyperlink{group___s_i_g_n_a_l_s_ga19345763f88f56f70252c7a97d831351}{signals\+\_\+data\+\_\+type}} and \mbox{\hyperlink{group___s_i_g_n_a_l_s_gaa8356b463bcf4f2de5fb8ab61004eb5a}{signals\+\_\+signal\+\_\+type}}. Signals can be chained together. For Example, four Signals of Type SIGNALS\+\_\+\+CAN\+\_\+\+MESSAGE could recieve four Temperatures, wich would then be interpreted to float Values using thier respective Callback Functions. Another Signal of Type SIGNALS\+\_\+\+INTERNAL\+\_\+\+SIGNAL could then find the highest of these Temperatures using its Callback Function. To complete the Chain, a Signal of Type SIGNALS\+\_\+\+DISPLAY\+\_\+\+SIGNAL can be used to convert the float value of the previous Maximising Function to a String Command wich can be interpreted by a Display.
